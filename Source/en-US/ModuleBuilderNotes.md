# en-US/

The locale-specific help folder for English (US). PowerShell's `Get-Help`
system looks for external help files here at runtime.

Add this folder to `CopyPaths` in `Build.psd1` when MAML help is adopted.
Comment-based help inside each function is the simpler alternative.

## Standard files

| File | Purpose |
|------|---------|
| `<ModuleName>-Help.xml` | MAML help file. Generated by PlatyPS (`New-ExternalHelp`) from Markdown command docs. |
| `about_<ModuleName>.help.txt` | `about_` topic shown by `Get-Help about_<ModuleName>`. Plain text, hard-wrapped at 80 chars. |

## Workflow (if adopted)

1. Author function docs as comment-based help in each `.ps1` file.
2. Generate Markdown stubs with `New-MarkdownHelp -Module <ModuleName>`.
3. Run `New-ExternalHelp -Path ./docs/commands -OutputPath ./Source/en-US` to
   produce `<ModuleName>-Help.xml`.
4. Add `'./en-US'` to `CopyPaths` in `Build.psd1` so ModuleBuilder copies it
   to the output folder.
